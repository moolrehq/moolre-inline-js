Basic Integration

Standard Checkout

Use popup.checkout() to trigger a secure payment popup. Perfect for simple one-time payments — with automatic event handling. 


// Include Moolre script
// <script src="https://moolre-inline-js.vercel.app/v1/inline.js"></script>

const popup = new MoolrePay();

document.getElementById("checkoutBtn").addEventListener("click", () => {
  popup.checkout({
    publicKey: "MOOLRE_PUBLIC_KEY",
    accountNumber: "MOOLRE_ACCOUNT_NUMBER",
    amount: 50,
    currency: "GHS",
    email: "customer@moolre.com",
    externalRef: "trx_" + Date.now(),
    onLoad: () => console.log("💡 Checkout loaded"),
    onSuccess: (res) => console.log("✅ Payment success:", res),
    onCancel: () => console.log("❌ Payment cancelled"),
    onError: (err) => console.error("⚠️ Payment error:", err),
    onClose: () => console.log("👋 Checkout closed"),
  });
});

Preload Transaction

Use popup.preloadTransaction() to load the payment popup in the background — so when the user clicks “Pay”, it opens instantly. 


const popup = new MoolrePay();
let openPayment = null;

document.getElementById("preloadBtn").addEventListener("click", async () => {
  try {
    openPayment = await popup.preloadTransaction({
      publicKey: "MOOLRE_PUBLIC_KEY",
      accountNumber: "MOOLRE_ACCOUNT_NUMBER",
      amount: 3,
      email: "customer@moolre.com",
      currency: "GHS",
      externalRef: "trx_" + Date.now(),
      onLoad: () => console.log("💡 Checkout loaded (preload)"),
      onSuccess: (res) => console.log("✅ Payment success:", res),
      onCancel: () => console.log("❌ Payment cancelled"),
      onError: (err) => console.error("⚠️ Error:", err),
      onClose: () => console.log("👋 Checkout closed"),
    });

    document.getElementById("openPreloadedBtn").disabled = false;
    console.log("✅ Preload successful!");
  } catch (err) {
    console.error("❌ Preload failed:", err);
  }
});

document.getElementById("openPreloadedBtn").addEventListener("click", () => {
  if (typeof openPayment === "function") openPayment();
});
🧠 Developer Reference

Available Event Callbacks

Every Moolre checkout supports event callbacks for fine-grained control:

onLoad: Fired when the checkout iframe fully loads.

onSuccess: Triggered when payment completes successfully.

onCancel: Runs when the user closes or cancels checkout.

onError: Captures failed transactions or API errors.

onClose: Called after the modal completely closes.
